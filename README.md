# 🛍 ربات مدیریت فروشگاه تلگرام

یک ربات کامل و حرفه‌ای برای مدیریت فروشگاه آنلاین در تلگرام، نوشته‌شده با **Python 3.13** و
**python-telegram-bot v21+**، با معماری Clean Architecture و اصول SOLID.

## ✨ ویژگی‌ها

### برای کاربران
- ثبت‌نام ساده با دریافت نام و شماره تماس
- مشاهده دسته‌بندی‌ها و محصولات به‌صورت صفحه‌بندی‌شده
- مشاهده جزئیات کامل محصول همراه با چند تصویر
- جستجوی محصول بر اساس نام و توضیحات
- سبد خرید تعاملی (افزودن، افزایش/کاهش تعداد، حذف)
- ثبت سفارش با دریافت آدرس و شماره تماس
- مشاهده تاریخچه سفارش‌ها و وضعیت هرکدام
- پروفایل کاربری و اطلاعات پشتیبانی

### برای مدیران (ادمین)
- پنل مدیریت اختصاصی و محافظت‌شده
- افزودن، ویرایش، حذف و فعال/غیرفعال‌سازی محصولات
- مدیریت چند تصویر برای هر محصول
- مدیریت قیمت و موجودی محصولات به‌صورت مستقل
- مدیریت دسته‌بندی‌ها (افزودن، ویرایش، حذف)
- مدیریت کامل سفارش‌ها و تغییر وضعیت (در انتظار، تایید شده، در حال ارسال و ...)
- آمار فروش، پرفروش‌ترین محصولات، محصولات با موجودی کم
- ارسال پیام همگانی (Broadcast) به تمام کاربران
- مدیریت کاربران (مسدودسازی/رفع مسدودیت)
- گزارش‌گیری از لاگ فعالیت‌های سیستم

### فنی
- معماری کاملاً ماژولار (Handlers → Services → Repositories → Database)
- رعایت اصول SOLID و Clean Code
- پایگاه داده SQLite با دسترسی ناهمزمان (aiosqlite)
- اعتبارسنجی کامل ورودی‌های کاربر
- مدیریت خطای سراسری و جلوگیری از کرش ربات
- ثبت لاگ کامل در فایل و پایگاه داده
- تنظیمات از طریق فایل `.env`
- تست‌های واحد با pytest

## ⚙️ تنظیم فایل .env

فایل نمونه `.env.example` را کپی کرده و به `.env` تغییر نام دهید:

```bash
cp .env.example .env
```

سپس فایل `.env` را باز کرده و مقادیر زیر را با اطلاعات واقعی خودتان جایگزین کنید:

| متغیر | توضیح | نمونه |
|---|---|---|
| `BOT_TOKEN` | توکن ربات دریافتی از BotFather | `123456:AAExxxxxx` |
| `ADMIN_IDS` | شناسه عددی ادمین‌ها، جدا شده با کاما | `111111111,222222222` |
| `DATABASE_PATH` | مسیر فایل دیتابیس SQLite | `database/shop.db` |
| `LOG_LEVEL` | سطح لاگ‌گیری | `INFO` |
| `LOG_FILE` | مسیر فایل لاگ | `logs/bot.log` |
| `CURRENCY` | واحد پول نمایشی | `تومان` |
| `ITEMS_PER_PAGE` | تعداد آیتم در هر صفحه لیست | `6` |
| `MAX_CART_QUANTITY` | حداکثر تعداد هر کالا در سبد خرید | `99` |
| `SUPPORT_USERNAME` | آیدی تلگرام پشتیبانی (بدون @) | `your_support_id` |
| `SHOP_NAME` | نام فروشگاه | `فروشگاه آنلاین من` |

> ⚠️ **مهم:** فایل `.env` هرگز نباید در Git Commit یا به‌صورت عمومی منتشر شود. این فایل در `.gitignore` نادیده گرفته شده است.

## ▶️ اجرای پروژه

پس از تنظیم فایل `.env`، ربات را اجرا کنید:

```bash
python bot.py
```

در اولین اجرا، پایگاه داده و جداول موردنیاز به‌صورت خودکار در مسیر تعیین‌شده ساخته می‌شوند.
سپس به ربات خود در تلگرام پیام `/start` را ارسال کنید تا فرآیند ثبت‌نام شروع شود.

اگر شناسه تلگرام شما در متغیر `ADMIN_IDS` قرار داشته باشد، پس از ثبت‌نام دکمه
«⚙️ پنل مدیریت» در منوی اصلی شما ظاهر خواهد شد.

## 🐳 اجرا با Docker

### با Docker Compose (پیشنهادی)

```bash
docker compose up --build -d
```

این دستور ایمیج را می‌سازد، ربات را در پس‌زمینه اجرا می‌کند و پوشه‌های `database/` و
`logs/` را به‌صورت Volume به کانتینر متصل می‌کند تا داده‌ها پس از ری‌استارت از بین نروند.

برای مشاهده لاگ‌های زنده:

```bash
docker compose logs -f
```

برای توقف:

```bash
docker compose down
```

### با Docker به‌صورت مستقیم

```bash
docker build -t telegram-shop-bot .
docker run -d --name shop-bot --env-file .env \
    -v $(pwd)/database:/app/database \
    -v $(pwd)/logs:/app/logs \
    telegram-shop-bot
```

## 🧪 اجرای تست‌ها

ابتدا وابستگی‌های تست را نصب کنید:

```bash
pip install pytest pytest-asyncio
```

سپس تست‌ها را اجرا کنید:

```bash
pytest -v
```

تست‌ها شامل اعتبارسنجی ورودی‌ها و تست‌های یکپارچگی (Integration) برای سرویس‌های محصول و
سبد خرید با استفاده از پایگاه داده SQLite در حافظه (`:memory:`) هستند.

## 📖 راهنمای استفاده

### برای کاربر عادی

1. ربات را با `/start` شروع کنید و مراحل ثبت‌نام (نام و شماره تماس) را کامل کنید.
2. از دکمه «🛍 مشاهده محصولات» دسته‌بندی و سپس محصول مورد نظر را انتخاب کنید.
3. با دکمه «🛒 افزودن به سبد خرید» محصول را به سبد اضافه کنید.
4. از دکمه «🛒 سبد خرید» تعداد را تنظیم کرده و روی «✅ ثبت سفارش» بزنید.
5. آدرس و شماره تماس را وارد کرده و سفارش را نهایی کنید.
6. وضعیت سفارش‌های خود را از «📦 سفارش‌های من» پیگیری کنید.

### برای ادمین

1. پس از ثبت‌نام (با شناسه‌ای که در `ADMIN_IDS` قرار دارد)، دکمه «⚙️ پنل مدیریت» ظاهر می‌شود.
2. از «📦 مدیریت محصولات» می‌توانید محصول جدید اضافه یا محصولات موجود را مدیریت کنید.
3. از «🗂 مدیریت دسته‌بندی‌ها» دسته‌بندی‌های فروشگاه را مدیریت کنید.
4. از «🧾 مدیریت سفارش‌ها» سفارش‌های ثبت‌شده را مشاهده و وضعیت آن‌ها را تغییر دهید.
5. از «📊 آمار فروش» گزارش کلی فروشگاه (درآمد، پرفروش‌ترین‌ها، موجودی کم) را ببینید.
6. از «📢 ارسال پیام همگانی» می‌توانید به تمام کاربران فروشگاه اطلاع‌رسانی کنید.

## 🏗 معماری پروژه

پروژه بر اساس لایه‌بندی زیر طراحی شده تا افزودن قابلیت‌های جدید در آینده ساده و بی‌خطر باشد:

```
Handlers (تلگرام)  →  Services (منطق کسب‌وکار)  →  Repositories (دسترسی به داده)  →  SQLite
```

- **Handlers** فقط مسئول دریافت Update از تلگرام، فراخوانی سرویس مناسب و نمایش پاسخ هستند.
- **Services** تمام منطق کسب‌وکار (اعتبارسنجی قوانین، محاسبات، هماهنگی بین Repositoryها) را در خود دارند.
- **Repositories** تنها مسئول اجرای کوئری‌های SQL و تبدیل نتیجه به مدل‌های داده هستند.
- **Models** موجودیت‌های ساده و بدون منطق (Data Classes) هستند.

این جداسازی مطابق اصل **Single Responsibility** از SOLID، تست‌پذیری بالا و توسعه آسان در
آینده (مثلاً افزودن درگاه پرداخت، پنل وب یا API) را فراهم می‌کند.
